🚀 Projeto de Testes Manuais: SwagLabs (SauceDemo)
📋 Sobre o Projeto
Este repositório contém a documentação de testes funcionais para a loja virtual SwagLabs, desenvolvida como parte do desafio de projeto da DIO (Digital Innovation One). O objetivo foi aplicar na prática as atividades do dia a dia de um QA em um ambiente ágil.

🛠️ Ferramentas Utilizadas
Jira Software: Gerenciamento do backlog e das User Stories.

Xray: Gestão de casos de teste (Step-by-Step e BDD).

MindMup / Miro: Criação de Mapas Mentais para levantamento de cenários.

📂 Documentação Disponível (PDFs)
Neste repositório, você encontrará os seguintes documentos detalhados:

User Stories: Documentação das histórias de usuário (Login e Checkout) com foco em valor de negócio e critérios de aceite.

Mapa Mental: Planejamento visual dos fluxos de teste.

Casos de Teste (Manual): Escrita detalhada utilizando a técnica Step-by-Step.

Casos de Teste (BDD): Escrita baseada em comportamento utilizando a sintaxe Gherkin (Dado, Quando, Então).

⚙️ Metodologia Aplicada
Durante o desenvolvimento, segui as seguintes etapas:

Análise de Requisitos: Entendimento das regras de negócio da loja SwagLabs.

Planejamento: Identificação de caminhos felizes e fluxos de exceção (ex: validação de campos obrigatórios).

Documentação: Criação de artefatos que garantem a rastreabilidade entre os requisitos (User Stories) e os testes.
